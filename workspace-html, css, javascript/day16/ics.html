모듈별 주요기능

컨텐츠매니저

서비스 될 컨텐츠의 생성, 수정, 삭제 등의 작업 및 사전 정의된 승인 룰에 의해 해당 컨텐츠의 생성에서 승인에 이르는 일련의 과정을 관리합니다.

사이트매니저

컨텐츠 매니저에 등록되어 승인을 득한 컨텐츠들을 해당 서비스 카테고리와 디자인 템플릿에 배치하고, 실제 물리적 서버에 구성되는 디렉토리 및 서비스 카테고리를 관리합니다.

운영매니저

컨텐츠가 서비스 될 사이트 생성 및 컨텐츠를 배포할 호스트 정보 등을 관리하고 아울러 전체 작업자들의 작업 내역 및 제반 정보를 모니터링 합니다.

어드민매니저

서비스 될 컨텐츠의 생성, 수정, 삭제 등의 작업 및 사전 정의된 승인 룰에 의해 해당 컨텐츠의 생성에서 승인에 이르는 일련의 과정을 관리합니다.

RESTful API를 통한 기능 연계

ICS는 CMS 내에서 생산 관리하는 컨텐츠의 일반 배포 서비스를 비롯하여 내/외부 시스템 간의 컨텐츠 연동을 통한 허브로서의 역할까지 대응이 가능합니다.




특장점

No DB connection
컨텐츠와 템플릿, 개발코드의 조합을 통해서 컨텐츠 파일을 생성하며, DB와 직접 접속하지 않는 방식으로 파일 배포

안전한 파일 배포 서비스
CMS에서 등록된 컨텐츠는 서비스 서버 측에 설치한 CMS 리시버 프로그램을 통해 파일을 암호화 처리하여 안전하게 전송

동적 메타필드
컨텐츠를 구성하는 부가정보(메타데이터)는 컨텐츠에 따라 증감 될 수 있으므로, 관리자가 손쉽게 추가하거나, 삭제, 변경하는 등 동적으로 메타데이터 필드를 관리 가능





통합 사이트 관리 환경
기존의 백오피스를 통한 웹사이트 운영 관리에서 벗어나, 여러 개의 사이트를 통합 운영 관리를 할수 있는 구조로, 사이트 별 플랫폼(Java, .NET, node.js 등)이 다르더라도 각각에 맞는 파일 등을 관리하고, 권한을 각각 부여

비주얼 웹에디터
크로스브라우징을 지원하는 나모 크로스에디터를 내장하여, 쉽게 웹 콘텐츠를 제작하거나, HTML 제작을 할 수 있도록 지원

분산 처리 환경
시스템 부하를 고려하여, 어플리케이션의 병렬 증축이 가능한 분산 처리 환경을 지원





동적 컨텐츠 서비스
정적 컨텐츠 뿐만 아니라, 개인화 서비스 및 추천 서비스 지원을 위해 동적 컨텐츠 서비스를 제공

표준 연동 API 제공
다양한 어플리케이션과의 연계를 위해 표준 연동을 위한 Restful API를 지원하고 있으며, 이를 활용하여 CRM, ERP, KMS 등 다양한 시스템과 연결 가능

다국어 서비스 최적화
UTF-8 캐릭터 셋을 기본으로 한 기반 환경으로, 다국어 서비스 및 UI 내의 모든 언어 로케일 또한 영문/중문/일문/국문/인도네시아어를 지원
